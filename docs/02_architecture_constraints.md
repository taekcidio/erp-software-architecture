# 2. Restricciones de arquitectura

## 2.1 Decisiones tecnológicas del taller
- **Frontend:** SPA Web con React.
- **Backend:** API monolítica en Java con Spring Boot.
- **Base de datos:** PostgreSQL.
- **Integraciones externas (conceptual):** Sistema contable (API/JSON) y servicio de correo (SMTP/API).
- **Diagramación:** PlantUML.
- **Repositorio:** GitHub público para alojar documentación arc42 y diagramas.

## 2.2 Restricciones y supuestos
- Para el taller se usa una arquitectura simple y plausible, suficiente para documentar C4 + UML.
- No se implementa código ejecutable en este taller; el foco es documentación de arquitectura.
